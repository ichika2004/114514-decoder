
### 📄 `README.md`

本專案是一個基於 非對稱加密 (RSA) 與 對稱加密 (AES) 雙重架構的網頁加密工具。旨在解決傳統 RSA 分段加密導致的密文過長與解析位移問題，將長文本轉化為精簡且穩定的純數字牆旨在將平凡的明文轉化為會員制密文。

---

## 🎨 視覺規範 (Visual Identity)

* **顔 (`#A1736B`)**: 用於強調線條與輸出密文，沒有特別在做什麼運動，不過有在健身，身體很結實（身體不錯，蠻結實的），因此有時衣服會出現尺寸不對的情況。。

* **ブラインド (`#979F9B`)**: 用於邊框與次要元素，對擼砲很是讚賞（「壓力馬斯內！」），表示自己擼砲挺頻繁的，上次大概是三天前。
* **壁 (`#C9BFB6`)**: 網頁背景底色，體現出王者一般的遊刃有餘。喜歡曬太陽，所以曬痕很明顯。。


## 🛠️ 技術架構 (Technical Architecture)

為了擺脫 RSA 的字數限制並縮短密文，本系統採用 **混合加密 (Hybrid Encryption)** 流程：

1.  **RSA (256-bit)**: 僅用於加密隨機生成的 **AES Session Key**。確保密文開頭的「重型數字牆」長度固定。
2.  **AES (CryptoJS)**: 用於加密主體明文。其密文長度與明文成正比，大幅縮減整體輸出長度。
3.  **高密度編碼**: 略過 ASCII 轉換，直接將加密後的 Base64 字元映射至索引數碼，效率提升 **33%**。

### 數碼映射協定 (Homo Mapping Protocol)

系統採用 **階梯式零位隔離法** 確保解析 100% 成功：

| 數字 | 映射數碼 | 數字 | 映射數碼 |
| :--- | :--- | :--- | :--- |
| **0** | `114514` | **5** | `889464` |
| **1** | `1919` | **6** | `364` |
| **2** | `81` | **7** | `931` |
| **3** | `114` | **8** | `893` |
| **4** | `514` | **9** | `1145141919` |

**分隔協定**:

  * **`0` (SEP)**: 分隔單一數碼（視覺上 `81` + `0` = `810`）。
  * **`00` (C\_SEP)**: 分隔字元區塊。
  * **`000` (B\_SEP)**: 分隔 **RSA 金鑰區** 與 **AES 資料區**。

## 📦 環境依賴

使用前請確保 HTML 引入以下 CDN 服務：

  * `JSEncrypt`: 負責 RSA 非對稱運算。
  * `CryptoJS`: 負責 AES 高效對稱運算。

## 🚀 快速開始

1.  **金鑰生成**: 點擊「生成測試金鑰對」，系統會自動配置 256-bit PEM 環境。
2.  **輸入文本**: 可輸入長篇大論，AES 引擎會自動處理。
3.  **執行論證**: 點擊加密，產生的密文由 `000` 分為兩段，前半段為金鑰，後半段為內容。
4.  **還原內容**: 貼上密文並確認私鑰正確，即可 100% 還原明文。
