
### 📄 `README.md`

本專案是一個基於 **RSA 非對稱加密演算法** 與 **114514 數碼映射（Digit Mapping）** 的網頁加密工具。旨在將平凡的明文轉化為會員制密文。

## 🛠️ 技術核心 (Technical Core)

### 1. 加密演算法
* **核心庫**: `JSEncrypt` (JavaScript RSA 實作)。
* **金鑰長度**: **256-bit**。
    * *註：此長度旨在平衡金鑰的精簡度與實用性。單次加密建議長度為 20 個字元內。*
* **格式**: 標準 **PEM** 格式（包含 `-----BEGIN...-----` 前後綴），確保與主流資安工具相容。

### 2. 純數字論證編碼 (Homo Mapping)
為了達成「純數字密文」且避免歧義，本系統採用 **「0 分隔拆解法」**。
每個 ASCII 字元會被補足為三位數（如 `A` → `065`），隨後根據下表進行數碼映射：

| 數字 | 映射數碼 (HOMO_MAP) | 數字 | 映射數碼 (HOMO_MAP) |
| :--- | :--- | :--- | :--- |
| **0** | `114514` | **5** | `191981` |
| **1** | `1919` | **6** | `811919` |
| **2** | `81` | **7** | `5141919` |
| **3** | `114` | **8** | `1145141919` |
| **4** | `514` | **9** | `114514191981` |

**分隔符號**: 使用 `0` 作為數碼間的分隔，確保解碼時能精確切割，徹底解決 `1+2=5` 的識別衝突。

## 🚀 快速上手 (Quick Start)

1.  **生成金鑰**: 點擊「生成金鑰」按鈕，系統將自動產生會員制 256-bit 公私鑰對，並同步填入加密與解密專區。
2.  **進行加密**: 在加密區輸入明文，點擊「進行惡臭論證」，下方將產生純數字牆。
3.  **資料傳遞**: 點擊「一鍵填入解密框」，密文將自動傳輸至解密專區。
4.  **還原明文**: 確認解密私鑰正確後，點擊「還原明文」即可看見原始資料。
